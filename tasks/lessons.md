## UI / UX

### 絵文字は Text() 内で文字化けする場合がある
- **問題**: `Text("🕐")` がデバイス上で「?」と表示された
- **原因**: 一部の絵文字は SF Symbols やフォントとの互換性で正しくレンダリングされない
- **対策**: UI テキスト内の絵文字は日本語テキスト（「時計ボタン」等）に置き換えるか、`Image(systemName:)` を使う

### 機能名は直感的に
- **問題**: 「バージョン管理」→「保存履歴」→「バックアップ」と 3 回変更が発生
- **教訓**: 技術用語ではなく、ユーザーが直感的に理解できる名前を最初から選ぶ。「バージョン管理」は開発者目線、「バックアップ」はユーザー目線

### データ削除は関連データもすべて含める
- **問題**: 「すべてのデータを削除」でバックアップが残ったままだった
- **対策**: 削除系の操作を実装する際は、関連するすべてのデータ（バックアップ、設定、ボード名等）を網羅的にリストアップしてから実装する
- **対策 2**: アラートメッセージに何が消えるか明示する

## テスト

### UI テストで multiline Text の検索に注意
- **問題**: `Text("保存された\nバージョンはありません")` を `staticTexts` で完全一致検索しても見つからない
- **原因**: SwiftUI は `\n` で分割された Text を別々の staticText 要素として認識する場合がある
- **対策**: `NSPredicate(format: "label CONTAINS %@", "部分文字列")` で部分一致検索する

### UI テストの状態依存を避ける
- **問題**: テスト順序によって「空メッセージ表示テスト」が別テストの保存データに影響された
- **対策**: 状態に依存するテストは避けるか、`setUp` で確実にリセットする。状態非依存のテスト（ボタン存在確認、タップ可能性等）を優先する

### シミュレータの preflight check 失敗
- **問題**: `Application failed preflight checks` でテストが起動しない
- **対策**: `xcrun simctl shutdown all` → `xcrun simctl erase <device>` → リトライ

## ローカリゼーション

### String Catalog (.xcstrings) を使う
- **方針**: Xcode 15+ では `.strings` / `.stringsdict` ではなく `.xcstrings` (String Catalog) を使用
- **利点**: JSON 形式で差分管理しやすい、Xcode 上で一覧管理可能
- **注意**: `LocalizedStringKey` は `Text()` / `Button()` で自動適用。プログラム内文字列は `String(localized:)` を使う

### フォーマット指定子の保持
- **注意**: `%lld`, `%@` 等のフォーマット指定子は全言語で必ず保持する
- **注意 2**: 改行 `\n` も全言語で位置を意識して保持する

## アーキテクチャ

### Widget / Watch の Shared コード
- **構成**: `WidgetMemo/Shared/` を Widget Extension と Watch アプリの両方で共有
- **注意**: `project.yml` で `sources` に Shared パスを追加する際、同一ファイルが複数グループに属する warning が出るが動作に影響なし
